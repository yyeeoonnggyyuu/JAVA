--------정리--------

1. 기본 타입 (Primitive Type)
저장 방식: 변수에 값 자체가 저장됩니다.
메모리 위치: 기본 타입 변수는 스택(Stack) 메모리에 저장됩니다.
값 비교: 두 변수를 비교할 때는 값 자체를 비교합니다.
종류: Java에는 8가지 기본 타입이 있습니다.
byte, short, int, long: 정수형
float, double: 실수형
char: 문자형
boolean: 논리형
기본값: 기본 타입 변수는 명시적으로 초기화하지 않으면 기본값을 가집니다. 예를 들어 int는 0, boolean은 false입니다.
2. 참조 타입 (Reference Type)
저장 방식: 변수에 값이 아닌, 객체의 주소(참조)가 저장됩니다.
메모리 위치: 객체는 힙(Heap) 메모리에 저장되고, 변수는 스택에 객체의 주소를 가집니다.
값 비교: 두 변수를 비교할 때는 객체의 주소(참조)를 비교합니다. 객체의 실제 값을 비교하려면 equals() 메서드를 사용해야 합니다.
종류: 클래스, 배열, 인터페이스 등이 참조 타입에 속합니다. 예를 들어 String, Array, 사용자 정의 클래스 등이 있습니다.
기본값: 참조 타입 변수는 초기화되지 않으면 null 값을 가집니다.
주요 차이점 정리
구분	기본 타입	참조 타입
저장 값	값 자체	객체의 참조(주소)
메모리	스택 메모리	힙 메모리(객체), 스택 메모리(참조)
비교	값 비교	주소(참조) 비교 (equals()로 값 비교 가능)
기본값	타입별 기본값 존재 (0, false 등)	null
기본 타입은 단순한 데이터를 다루고, 참조 타입은 복잡한 객체나 데이터를 다룰 때 사용됩니다.
